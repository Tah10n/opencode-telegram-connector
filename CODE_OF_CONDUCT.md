# Code of Conduct

This project aims to be practical, respectful, and easy to collaborate on.

## Expected behavior

- Be respectful and constructive.
- Assume good intent, but be clear and specific when reporting problems.
- Keep feedback focused on the code, behavior, and user impact.
- Respect maintainer and contributor time: keep issues and pull requests scoped and reproducible.
- Protect private data: never post tokens, passwords, private state files, or other sensitive runtime data in public threads.

## Unacceptable behavior

- Harassment, hate speech, discrimination, threats, or personal attacks.
- Trolling, deliberate disruption, or repeated bad-faith arguments.
- Publishing someone else's private information without permission.
- Sharing secrets, credentials, or sensitive operational details in public issues or pull requests.
- Spam, off-topic promotion, or intentionally misleading reports.

## Scope

This policy applies to repository issues, pull requests, reviews, and other project spaces where contributors interact around this codebase.

## Enforcement

Project maintainers may edit or remove content, close threads, reject contributions, or block participants whose behavior violates this policy.

## Reporting

Please report conduct problems privately to the maintainer through an existing private contact channel when one is available.

Until a dedicated contact address is added, you may open a minimal public issue asking for a maintainer contact path. Do not include personal details, sensitive context, or allegations in that issue. If a report overlaps with a security concern, follow the guidance in `SECURITY.md`.
